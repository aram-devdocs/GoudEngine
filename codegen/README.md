# GoudEngine Codegen Pipeline

Generates SDK wrapper code for C#, Python, and TypeScript from a single schema.
One source of truth, four generators, zero hand-written SDK boilerplate.

## Quick Start

```bash
./codegen.sh
```

This runs the full pipeline: build Rust, lint layers, validate coverage,
generate all SDKs, and validate schema consistency.

## Running Individual Generators

Build the Rust crate first (generates `ffi_manifest.json`):

```bash
cargo build
```

Then run any generator:

```bash
python3 codegen/gen_csharp.py      # C# SDK
python3 codegen/gen_python.py      # Python SDK
python3 codegen/gen_ts_node.py     # TypeScript Node.js SDK
python3 codegen/gen_ts_web.py      # TypeScript Web (WASM) SDK
```

## Schema Reference

### goud_sdk.schema.json

The universal SDK definition. Contains:

- `types` -- value types (Color, Vec2, Transform2D) with fields and factories
- `enums` -- enumeration types (Key, MouseButton) with numeric values
- `tools` -- high-level objects (GoudGame) with constructors, destructors, and methods

### ffi_mapping.json

Maps schema methods to C ABI function names and parameter signatures.
Hand-maintained. Must stay in sync with the schema.

### ffi_manifest.json

Auto-generated by `cargo build` via the `#[goud_api]` proc macro. Lists every
`#[no_mangle] extern "C"` function extracted from Rust source. Do not edit.

## Validation

```bash
python3 codegen/validate_coverage.py   # Every FFI function has a mapping
python3 codegen/validate.py            # Schema-mapping consistency
```

Both validators run automatically as part of `./codegen.sh` and will exit
with an error if any check fails.

## Shared Utilities

`sdk_common.py` provides schema loading, name converters (PascalCase, snake_case,
camelCase, SCREAMING_SNAKE), and type mapping tables for all target languages.
All generators import from this module.
