# codegen/ — SDK Code Generation Pipeline

## Purpose

Generates SDK wrapper code for all target languages from a single source of truth.
One schema, one mapping, multiple outputs. No hand-written SDK boilerplate.

## Pipeline

```
goud_sdk.schema.json   (what the SDK API looks like)
        +
ffi_mapping.json       (how SDK methods map to FFI functions)
        +
ffi_manifest.json      (actual FFI functions from cargo build -- generated, do not edit)
        |
        v
   gen_<lang>.py        (language-specific generators)
        |
        v
   sdks/<lang>/generated/   (auto-generated SDK files)
```

## Key Files

- `goud_sdk.schema.json` — Universal SDK schema. Defines types, tools (classes), methods,
  parameters, return types, and factory methods. Every SDK is derived from this file.
- `ffi_mapping.json` — Maps schema tools/methods to `extern "C"` FFI function names,
  parameter types, and struct field layouts. Hand-maintained.
- `ffi_manifest.json` — Generated by `cargo build` (via the `#[goud_api]` proc macro).
  Lists every `#[no_mangle]` function with its actual Rust signature. Do not edit.
- `sdk_common.py` — Shared utilities: schema loader, mixin resolver, naming conventions
  (PascalCase, snake_case, camelCase), type mappings for all target languages.

## Generators

| Script | Output | Target |
|--------|--------|--------|
| `gen_csharp.py` | `sdks/csharp/generated/` | C# (.NET 8.0) |
| `gen_python.py` | `sdks/python/goud_engine/generated/` | Python (ctypes) |
| `gen_ts_node.py` | `sdks/typescript-node/generated/` | TypeScript (N-API) |
| `gen_ts_web.py` | `sdks/typescript-web/generated/` | TypeScript (wasm-bindgen) |

All generators import `sdk_common.py` for schema loading and type mapping.

## Validators

- `validate.py` — Checks that every method in the schema has a mapping in `ffi_mapping.json`
  and vice versa. Run after schema or mapping changes.
- `validate_coverage.py` — Checks that every FFI function in `ffi_manifest.json` has a
  corresponding entry in `ffi_mapping.json`. Also compares parameter signatures. Run
  after `cargo build` to catch drift.

## Adding a New SDK Language

1. Add type mappings to `sdk_common.py` (e.g., `SWIFT_TYPES = {...}`)
2. Create `gen_swift.py` using an existing generator as a template
3. The generator reads from `load_schema()` and `load_ffi_mapping()`
4. Output generated files to `sdks/<lang>/generated/`
5. Add validation coverage in CI

## Gotchas

- `ffi_manifest.json` is gitignored-ish -- it changes on every build. Treat it as a
  build artifact, not a source file.
- Schema supports mixins (`$ref` in method lists) resolved by `sdk_common.resolve_mixins()`.
- Type mappings in `sdk_common.py` must stay in sync with `ffi_mapping.json`'s `ffi_types`.
- Generated files have a header comment: "This file is AUTO-GENERATED... DO NOT EDIT."
  Respect it.
