# SDK Parity Check

Scan the FFI layer and verify that every exported function has corresponding wrappers in both the C# and Python SDKs.

## When to Use

Run after adding or modifying FFI functions, after SDK changes, or before releases to verify complete coverage.

## What It Checks

1. **FFI → C# parity**: Every `#[no_mangle] extern "C"` function in `goud_engine/src/ffi/` has a matching `DllImport` declaration in `sdks/GoudEngine/`
2. **FFI → Python parity**: Every FFI function has a matching ctypes declaration in `sdks/python/goud_engine/bindings.py`
3. **Signature matching**: Parameter types and return types are compatible across the boundary

## Manual Audit Process

### Step 1: Collect FFI Exports

Find all public FFI functions:

```bash
rg "#\[no_mangle\]" goud_engine/src/ffi/ --type rust -A 2
```

This produces a list of functions like:
```rust
#[no_mangle]
pub extern "C" fn create_context(...) -> *mut Context { ... }
```

### Step 2: Check C# Bindings

For each FFI function, verify a matching DllImport exists:

```bash
rg "DllImport.*create_context" sdks/GoudEngine/ --type cs
```

Also check `NativeMethods.g.cs` (auto-generated by csbindgen):

```bash
rg "create_context" sdks/GoudEngine/NativeMethods.g.cs
```

### Step 3: Check Python Bindings

For each FFI function, verify a matching ctypes declaration exists in `bindings.py`:

```bash
rg "create_context" sdks/python/goud_engine/bindings.py
```

### Step 4: Check for SDK-Only Functions

Look for SDK functions that don't correspond to any FFI export (potential logic-in-SDK violation):

```bash
# C# methods that might contain logic
rg "public .* \w+\(" sdks/GoudEngine/ --type cs -g '!NativeMethods*' -g '!obj/*'

# Python functions
rg "def " sdks/python/goud_engine/ --type py -g '!test_*'
```

## Type Mapping Reference

| Rust FFI Type | C# Type | Python ctypes |
|---------------|---------|---------------|
| `i32` | `int` | `c_int` |
| `u32` | `uint` | `c_uint` |
| `f32` | `float` | `c_float` |
| `f64` | `double` | `c_double` |
| `bool` | `bool` | `c_bool` |
| `*mut T` | `IntPtr` | `POINTER(T)` or `c_void_p` |
| `*const c_char` | `string` | `c_char_p` |
| `()` (void) | `void` | `None` |
